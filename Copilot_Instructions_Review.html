Big Picture Architecture

Pipeline: Command Interface → QGED Authority Gate → Execution Layer → Evidence Record → Ledger Append → State Freeze

  • QGED (gate) and GCC (cycle) engines: QGED blocks unless SYSTEM_STATE, DOMAIN_STATE, AUTHORITY_STATE are all loaded.
  • Immutable ledger: append-only, hash-chained, never modify or delete entries.
  • Evidence record for every executed command.

AI Agent Constraints

  • YAI may only: diagnose, analyze, propose
  • Blocked from: financial execution, authority override, policy modification
  • Source: Governance Operating System/ai/llm_governance_context.json

Command & Ledger Schema

  • All commands: Governance Operating System/schemas/command_schema.json
  • Required fields: command_id, actor, action, complexity_score, risk_level, signer, payload, gate_decision
  • Action types: GO (execute), HOLD (pause), DIAGNOSE (analyze only)
  • Ledger: append-only, hash-chained, see Governance Operating System/schemas/ledger_schema.json

Evidence Metadata

  • Every executed command produces an evidence record (Evidence Metadata Extension.json template)
  • Fields: event_id, command, actor, domain, authority_validated, system_state, domain_risk_tier, timestamp, hash

Domains & Runtime Modes

  • Domains: CARSHUNTER (automotive, risk_tier 2, threshold 100,000), BPB (finance/advisory), IBN_SINA (medical, baseline complexity 6)
  • Each domain: domain_status, risk_tier, authority_required, transaction_threshold, compliance_mode
  • Runtime modes: development, operational, audit, freeze; risk levels: normal, elevated, critical
  • freeze_state: true → no new commands may be executed under any authority level

  • .github/AUTHORITY_STATE.json — live authority mode

Entry N

entry_id
command_id
