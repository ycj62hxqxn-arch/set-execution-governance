Architecture Layers

1. Decision Input Layer
AI agents or human operators submit proposed actions.

2. Admissibility Layer
Validates whether the proposed action is admissible under system rules.

3. Authority Verification Layer
Confirms that the actor or token initiating the request holds the required authority.

4. Execution Gate
Determines whether the action is allowed to mutate system state.

5. Evidence Freeze
Logs and freezes decision metadata for audit and verification.

6. System State Engine
Updates the operational state of the governance system.

7. Reporting Layer
Provides introspective system reporting via API.
