## 1. Diagnosed Issues

### A. Word (.docx) Export Error
- Error: "Word experienced an error trying to open the file."
- Possible causes:
  - File not generated in true DOCX format (plain text with .docx extension)
  - Missing required XML structure for Word compatibility
  - File permissions or disk space unlikely root cause

### B. PDF Export Error
- Error: "The file ... could not be opened. It may be damaged or use a file format that Preview doesn’t recognise."
- Possible causes:
  - File not generated as a true PDF (plain text with .pdf extension)
  - No PDF structure or encoding present

### C. HTML Export/Rendering Issues
- HTML files exported from previous sessions (esp. via WhatsApp or cross-workspace) lose layout and style, rendering as plain text or whitepaper style
- Causes:
  - WhatsApp and some channels strip CSS, inline styles, or reformat HTML
  - Exported HTML may lack robust, portable styling or use unsupported features

## 2. Lessons Learned

- **Tool Selection:**
  - Automated tools sometimes select the wrong export method for DOCX/PDF, producing invalid files (plain text with wrong extension)
  - True DOCX requires Office Open XML structure; true PDF requires PDF encoding
  - HTML export is more robust but can lose style/layout when shared via messaging apps or across environments

- **Cross-Environment Consistency:**
  - Exported files may render differently in different apps, OSes, or after being sent through channels like WhatsApp
  - HTML is safest for browser-based review, but not for regulatory or legal archiving unless style is preserved

- **Best Practices Going Forward:**
  - Use dedicated libraries/tools for DOCX (e.g., python-docx, pandoc) and PDF (e.g., ReportLab, wkhtmltopdf, browser print-to-PDF)
  - Always validate exported files in target environment before distribution
  - For compliance/archive, prefer PDF generated from browser print (ensures WYSIWYG)
  - For Word, use Office or Google Docs export, not just renaming .txt/.md
  - For HTML, use inline CSS and test in multiple browsers; avoid relying on external stylesheets

- **Operational Note:**
  - Add a technical review step to all protocol/document export workflows
  - Archive both the original markdown and a validated PDF/HTML version for audit
